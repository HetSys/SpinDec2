Getting Started

Download

To get the source code, the repository can be cloned to your chosen directory by typing the following command into a terminal (making sure you are in the correct directory):

$ git clone https://github.com/HetSys/SpinDec2

Installation

The code can be compiled with the installation script as follows:

$ ./install.sh -c

If the $PATH to the spindec binary isn't already in ~/.bashrc , a prompt will appear asking if the user wishes to add the /bin location to $PATH . The shell will then need to be reloaded using source ~/.bashrc for spindec to appear as executable.

Using the clean flag to remove compiled binaries

To recompile the code, or just to remove binaries from a previous compilation, the clean flag can be used:

$ ./install.sh -C

Running the code

After compilation of the code, the compiled binaries will be stored in the sub-directory /bin from the repository home. To execute the code, it is highly recommended to use the approach of creating a directory per calculation. This directory should contain an input file called input.txt which is discussed further here. The following syntax is used to run the computation in the directory where the input file is located:

$ mpirun -np n spindec

where n is the number of processors to run with.

  • $ ./spindec

This uses the parameters provided in the input file to run the simulation in serial and evolve the concentration grid over a the number of timesteps requested.

  • $ mpirun -np n ./spindec

When the code is compiled with MPI enabled, this command is used to run the code, where n is the number of processes requested (this will be further expanded upon in the examples).

  • $ python3 vis_spindec.py
