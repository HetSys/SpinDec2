Example - Running Serial With Constant Mobility

To run the code with a constant mobility field, the problem variable in your input.txt file needs to be set to "Constant". You must also make sure you set the variables mobility_a and mobility_b to the values you want to simulate. In our example, we shall set them to mobility_a = 4 and mobility_b = 4. For this example, the value of four is chosen such that \(M\) is equal to one when \(c_0 = 0.5 \). The forward Euler solver is very sensitive to the value of the input parameters, so some suggested values are given in the example input file picture below:


  concentration_max = 0.6
  concentration_min = 0.4
  domain_x_size = 128
  domain_y_size = 128
  mobility_a = 4
  mobility_b = 4
  free_energy_gradient_parameter = 0.0001
  bulk_free_energy = 1
  checkpoint_interval = 5000
  checkpoint_output_file = "out.cp"
  checkpoint_input_file = "out.cp"
  max_time = 4e-2
  time_step = 1e-6
  random_seed = 1234
  use_input = 0
  excitation_a = 10000
  excitation_b = 20000
  temperature_min = 950
  temperature_max = 1050
  problem = "constant"
  stabilization_term = 0
  write_frequency = 1
  single_precision = 0

The input file shows all the inputs that the user must specify, however only the mobility parameters will affect how \(M\) is calculated for this case.

Changing \(A\) and \(\kappa\)

In SpinDec2, you can change the bulk free energy scale factor \(A\) for the case where \(f(c) Ac^2(1-c)^2\). Interestingly, you should find that if you increase the scale factor and the free energy gradient parameter by a factor of 10, you should get the same result as the one above, but at a time, a tenth of that above. An example input file is shown below:


    concentration_max = 0.6
    concentration_min = 0.4
    domain_x_size = 128
    domain_y_size = 128
    mobility_a = 4
    mobility_b = 4
    free_energy_gradient_parameter = 0.001
    bulk_free_energy = 10
    checkpoint_interval = 5000
    checkpoint_output_file = "out.cp"
    checkpoint_input_file = "out.cp"
    max_time = 4e-3
    time_step = 1e-7
    random_seed = 1234
    use_input = 0
    excitation_a = 10000
    excitation_b = 20000
    temperature_min = 950
    temperature_max = 1050
    problem = "constant"
    stabilization_term = 0
    write_frequency = 1
    single_precision = 0

Changing The Initial Concentration Range

The starting initial concentration grid will affect the end state of the system. In the previous examples, we have used an initial range of 0.4 to 0.6, equating to a 50/50 mixture of the two species, as the initial average concentration \(c_0\) is 0.5. If we were to change this to be more biased towards species \(A\) (\(c_0 \approx 0.6\)) or species \(B\), (\(c_0 \approx 0.4\)), then we would expect to have less of a 50/50 split at the end of the simulation and have clumps of the less dominant element in a sea of the more dominant element. Defining a custom polynomial

In SpinDec2, you can define the bulk free energy polynomial yourself. This is done using the f(c) variable in the input file. The bulk free energy polynomial is crucial in defining how your binary alloy system will behave. In particular, the location of the minima will affect the range of values that \(c\) can take. Previously, we have used a polynomial where the minimas lie at \(c=0\) and \(c=1\), which is why the range in \(c\) the previous results have been between 0 and 1 in the final output. If we change the polynomial to be \(f(c) = c^4 - 2c^2 +1\), the minimums are now located at -1 and 1, with the local maxima (i.e the mixed state), now located at \(c = 0\) rather than 0.5. An example input file and expected results for this case are given below:


  concentration_max = 0.01
  concentration_min = 0.005
  domain_x_size = 128
  domain_y_size = 128
  mobility_a = 4
  mobility_a = 4
  free_energy_gradient_parameter = 0.0001
  bulk_free_energy = 1
  f(c) = 5,1.0,0,-2.0,0.0,1.0
  checkpoint_interval = 5000
  checkpoint_output_file = "out.cp"
  checkpoint_input_file = "out.cp"
  max_time = 3e-2
  time_step = 1e-6
  random_seed = 1234
  use_input = 0
  excitation_a = 10000
  excitation_b = 20000
  temperature_min = 950
  temperature_max = 1050
  problem = "constant"
  stabilization_term = 0
  write_frequency = 1
  single_precision = 0
