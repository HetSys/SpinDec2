Check-Pointing

The code writes a checkpoint file at regular (user-defined) intervals to a (user-defined) file. This is be done using a counter to measure the number of time steps elapsed after the previous checkpoint. Once this counter reaches the defined value, it resets, and a file is written containing the current state of the concentration and all metadata used to start the run. This file is written using the NetCDF external library. The user can input a checkpoint file to restart the code using the checkpoint_input_file variable in input.txt . The file read, and all data and metadata are stored in the appropriate variables. The code then continues as normal. The user-defined interval, checkpoint_interval , should be an integer greater than 0. This is checked on user input to avoid errors. The user-defined files are input in the form of a filename, set using checkpoint_output_file , of which the file extension is defined in the code. When reading from a checkpoint file, the code will catch any errors relating to reading it and then prompt the user if they wish to continue without the checkpoint or abort the run.
